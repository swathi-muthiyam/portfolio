    Platform: Scheduler - Device registration + Simulator (Instead of actual device) + device Integration


    Key Features:


    Device Registration (Bulk Ingestion):


    • A Function App automatically pulls bulk device data files from a designated SFTP location.
    • Each file contains metadata required to register multiple devices (e.g., device ID, type, location, etc.).
    • The Function App processes this data and registers the devices in the system.
    • Device registration is logged in a SQL database for tracking and auditing purposes.

      • Telemetry Reception:


    • Once registered, devices begin transmitting telemetry data (e.g., location, temperature, pressure).
    • This data is received in real time by a client system responsible for ingesting and interpreting the incoming signals.

      • Data Persistence:


    • Telemetry data is stored in Azure Cosmos DB, a scalable NoSQL database.
    • This structured storage allows for high-speed querying and flexible schema management.

      • KPI Population:


    • Data from Cosmos DB is used to compute and populate Key Performance Indicators (KPIs).
    • These KPIs support dashboards, alerts, and operational analytics, offering insights into device performance and status.
